Principle 7: Support top                - down processing (Perceptual)

 Display content/signals are perceived and interpreted based on past user
  experiences

 If format of display information conflicts with expectation, users seek   more
  physical evidence to guarantee signal is interpreted correctly

      Increases task time

 If display contains information that differs from expectations , users may see what
  they were expecting to see instead of what is really there .

      Increases errors

· Need for system activation
    creates expectancy for pilots that
    all systems should be active.

· Need to highlight those steps
    that deviate from expectation to
    ensure correct action.

Principle 8: Exploit redundancy gain (Perceptual)

 Presenting signal using more than one dimension of coding increases likelihood
  signal will be interpreted correctly

 Example: If user cannot tell what color traffic light is, they are still able to interpret
  information correctly by position of light.

      e.g., Auditory + haptic display - Cell phone ringtone plus vibration

Principle 9: Similarity will cause confusion (Perceptual)

 Make discriminable...
 Similar signals are likely to be confused
 Unnecessary and similar features should be removed from displays
 Dissimilar target features should be highlighted

       e.g., A423B9 is more similar to A423B8 than 8 is to 9

 Unnecessary similar features should be removed and dissimilar features should be highlighted

Principle 10: Knowledge in the world (Memory)

 A user should not need to retain important information solely in working memory or to
  retrieve it from long -term memory

 Example: explicit statements of what is to be done and when (flight procedure checklists)

 To much "knowledge in the world" = Visual clutter, complicated displays, difficult
  information retrieval

 Too much "knowledge in the head" = memory overload

 Need to provide good balance between knowledge in world (menus) and knowledge in
  head (commands)

 Help support understanding and orientation when moving between successive
  displays, pages, or "views"

 Memory is used to remember details from one display to be used on another
  display, this can be problematic

 Inspired by techniques used in film

      Use a fixed structure across multiple display elements (e.g., consistent layout of websites)

Principle 12: Principle of Predictive Aiding (Memory)

 Provide user with info on future events.
 Minimize load on "prospective memory" - recall

  something will happen in future.

      Displays eliminate demanding cognitive tasks and
        replace with simple perceptual tasks

 Prompt users to think about future events and
  allow for greater concentration on current task
  conditions

 Aid should be simple and intuitive in order to
  pose limited cognitive load

 Location and timing of displays should be such
  that users have enough time to react to identified
  event

      e.g., bridge height sign at bridge has little predictive
        utility

      e.g., tunnel height displays too far in advance of
        tunnel can lead to lack of user recall and failure to
        react to future event (tunnel collision)

Principle 13: Principle of Consistency (Memory)

 Based on task experience and long -term memory, people develop habits in using displays.
 Consistency refers to designing displays in manner that resembles other displays
 Familiar icons, actions and procedures from other displays will easily transfer to support

  processing of new displays if they are designed in a consistent manner
 e.g., make sure color -coding of your new display resembles color -coding of other existing

  displays for similar purpose
 e.g., make sure layout of controls/displays resembles layout of similar control panels

Principle 14: Principle of Pictorial Realism (Mental Model)

 A display should look like (i.e., be a picture of) system
  variable that it represents.
     e.g., Temperature has high and low values; therefore,
       thermometer should be oriented vertically.

 If system contains multiple elements, then display
  elements should be configured in manner that looks like
  how physical elements are configured in environment
